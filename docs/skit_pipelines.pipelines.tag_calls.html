skit_pipelines.pipelines.tag_calls package¶

Module contents¶

tag_calls(org_id: str, s3_path: str, data_label: str = '', job_ids: str = '', labelstudio_project_id: str = '', call_project_id: str = '', lang: str = '', notify: str = '', channel: str = '', slack_thread: str = '')[source]¶

A pipeline to upload a dataset for annotation.

Example payload to invoke via slack integrations:

@charon run tag_calls

{
    "org_id": 23,
    "job_ids": "1,2,3",
    "s3_path": "s3://bucket/path/to/file.csv"
}

To use labelstudio:

@charon run tag_calls

{
    "org_id": 23,
    "labelstudio_project_id": "41",
    "call_project_id": "100",
    "s3_path": "s3://bucket/path/to/file.csv",
    "data_label": "Client"
}
Parameters

  • org_id (str) – The organization id as per api-gateway.

  • job_ids (str) – The job ids as per tog. Optional if labestudio project id is provided.

  • labelstudio_project_id (str) – The labelstudio project id (this is a number) since this is optional, defaults to “”.

  • call_project_id (str) – The call project id (this is a number) since this is optional, defaults to “”.

  • lang (str, optional) – The language of the calls, defaults to “”

  • s3_path (str) – The s3 path to the dataset.

  • data_label (str, optional. Defaults to "Live") – A label to identify the source of a datapoint

  • notify (str, optional) – A comma separated list of slack ids: “@apples, @orange.fruit” etc, defaults to “”

  • channel (str, optional) – The slack channel to send the notification, defaults to “”

  • slack_thread (float, optional) – The slack thread to send the notification, defaults to “”
